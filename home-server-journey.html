Chapter 01

Where it Started

The whole thing started with a piece of software, not a hardware plan. I came across Unraid — an OS that provides RAID-style data protection but with a key twist: it works with any drives you have, regardless of size or brand. You don't need a matched set. You don't need to buy anything specific. You can literally use whatever is lying around.

That was it. That was the spark.

Why Unraid changed everything

Traditional RAID requires all drives to be identical — same size, same manufacturer, ideally same batch. If you want to expand, you're often rebuilding from scratch. Unraid throws all that out. Mix a 4TB, a 6TB, and an 8TB drive — no problem. Add another drive next year when you need more space — just add it. That flexibility is what makes it perfect for a home setup where you're growing organically, not provisioning a data center.

At the same time, I was building myself a new PC. That meant I had a perfectly functional motherboard, CPU, and RAM suddenly sitting idle. Combined with drives I'd pulled from an old Synology NAS and a few more rescued from work equipment headed for recycling, I had almost everything I needed already.

The actual out-of-pocket cost to get started was close to zero. Unraid made that possible — because the whole premise is that you use what you have. The software is what unlocks the value in hardware you'd otherwise throw away.

The lesson

Don't start by pricing out hardware. Start by understanding what Unraid can do with what you already own. You might be closer to a working server than you think.

There was one GPU decision worth mentioning: if you ever plan to transcode video for people outside your home, a dedicated Nvidia card can handle that work in hardware instead of burning CPU cycles. I added one early on. Whether you need it depends on how you plan to use Plex — more on that later.

Chapter 02

How Unraid Actually Works

The core idea in Unraid is simple: one drive is your parity drive, and all the others are data drives. If any single data drive fails, the parity drive can reconstruct what was on it. No data lost.

Example array — different sized drives, still protected

⚡Parity
12 TB
💾Data
8 TB
💾Data
6 TB
💾Data
4 TB
➕Add later
any size

The parity drive must be equal to or larger than your biggest data drive. Data drives can be any size. Add new drives anytime.

The "upgrade" rule

If you want to add a drive larger than your current parity drive, you need to buy two: one to become the new parity drive, and one to add as data. Or you can repurpose your current parity drive as a data drive once the new parity is in place. Either way — plan on buying in pairs when you scale up.

My personal strategy: once a year around Black Friday or Amazon Prime Day, I'd buy two matching large drives. One became the new parity. One went on the shelf as a hot spare — ready to swap in immediately if any drive failed. It costs more up front, but the peace of mind is worth it.

The motherboard limit you'll eventually hit

Most consumer motherboards only have 6 SATA ports — meaning 6 drives maximum before you need more. When you get there, the fix is a PCIe HBA card (Host Bus Adapter). A used LSI card on eBay runs $40–60 and adds 8 more SATA ports. Unraid treats them exactly like native ports — no reconfiguration needed. I had to do this myself once my array grew past what the motherboard could handle. It's a completely normal step in a growing build, not a problem.

My take

I bought the Lifetime license when they still called it "Pro." If you're building something you plan to run for years, Lifetime is the right call — the math works out fast. If you're just testing it, start with Starter. You can upgrade later and pay only the difference.

One important note: if you don't renew after year one, you still own the software. You just stop getting major updates. Security patches continue within your current minor version. It's not a hostage situation.

The point

Every one of these runs as an isolated Docker container. They don't conflict. They share the same hardware. You add them one at a time as you need them. Start with Plex. Add Pi-hole when you're curious. Add Home Assistant when you want to go deeper. There's no pressure to do it all at once.

Plex Library updates automatically. Requester gets notified when it's ready.

I run two instances of Radarr: the first grabs 1080p immediately, and a second instance syncs from the first and hunts for a 4K version. So most movies end up with both versions available without me doing anything.

On transcoding

Transcoding is when Plex converts a file on the fly because the client can't play the original format. It's CPU-heavy. The best approach: configure your library so files are already in a format all your devices can direct-play — no conversion needed. Tools like Tdarr can handle this pre-conversion automatically. The tradeoff is disk space versus CPU load during playback.

Two tools I haven't fully explored yet but are worth knowing about: Tdarr for automated pre-transcoding your library, and Wizarr for managing invitations to your Plex server.

Chapter 06

Why I Now Run Two Servers

My setup eventually split into two machines:

🗼

The Main Tower

Full-size case, many drives, all media storage. Runs Plex, the download stack, Unifi Network controller, and Pi-hole (primary instance).

📦

The NUC (Intel Next Unit of Computing)

Small single-drive box. Runs the reverse proxy (Nginx Proxy Manager) for anything that needs to be accessible from outside my home, plus Home Assistant as a VM, and Pi-hole (secondary instance for failover).

The reason to split them: the NUC handles all inbound internet traffic. If something's exposed to the public internet, it goes through there — not the main box. It also means I can restart or update the big server without killing Home Assistant.

On Pi-hole

Pi-hole acts as a DNS server for your whole home network. Every device routes DNS requests through it, and it blocks ad and tracker domains before they ever load. It's nearly entirely upside — faster browsing, fewer ads everywhere including apps. I run it on both boxes so if one goes down, the other picks up seamlessly.

Chapter 07

What I Actually Back Up (And What I Don't)

My philosophy evolved here. I used to think I needed to back everything up. Now I'm more surgical about it — and I rely entirely on cloud for the things that actually matter.

✓ Back this up

  • Photos (phone + old SLR library)
  • App & server configurations
  • Important documents
  • Music library (if irreplaceable)
  • Anything you can't recreate

↗ Don't bother

  • Plex media library
  • Downloaded movies & TV
  • Content you can re-download
  • Anything easily replaceable

For actual backup, I use iDrive for my computers (reasonably priced for the capacity) and native tools for phones. Cloud is the answer for anything irreplaceable — local backups help with quick recovery, but offsite is what protects you from fire or theft.

The honest truth about Plex media

I have terabytes of movies and TV shows I've never watched and probably never will. If it all burned tomorrow, I'd rebuild over time. Don't tie yourself in knots backing up content you can re-acquire. Be selective. Have a system for pruning things you'll never touch again.

Chapter 08

How This Thing Evolved Over Time

Started with salvaged parts

Spare case, recycled work drives, cheap motherboard. The point was to get it running, not to get it perfect.

Added Plex and a download stack

This became the core purpose: a media box for the household. Sonarr, Radarr, and a download client made it self-managing.

Outgrew the original case

Drives everywhere. Eventually bought a full-size tower with proper drive bays. Worth every dollar — the old solution was held together with cable ties.

Hit the motherboard's SATA limit — added an HBA card

Most motherboards only have 6 SATA ports. Once I filled those, I added a PCIe HBA (Host Bus Adapter) card — a used LSI card off eBay for around $40–50 — which added 8 more SATA ports instantly. Unraid sees them all as one seamless array. This is a completely normal part of growing a build and costs almost nothing.

Split off a NUC for networking duties

Home Assistant, reverse proxy, and Pi-hole moved to a small dedicated Intel NUC. Cleaner architecture, more resilient.

Simplified backups, dropped local music

Moved to cloud backup for everything that matters. Stopped hosting music locally — Spotify and Pandora handle 99% of listening now. The library still lives there but rarely gets touched.

Where it is now

A stable, mostly self-managing system. Light quarterly maintenance. Occasional new Docker container to explore. It runs in the background and mostly just works.

Chapter 09

Is Unraid the Only Option?

No. It's what I know and what I'd recommend for most people, but it's worth knowing what else exists.

🔵

Proxmox

Free, open-source, hypervisor-first. Great for running VMs. Steeper learning curve. Has come a long way — worth evaluating if you're comfortable with Linux. I tried it years ago and found Unraid friendlier, but that gap may have closed.

🟢

TrueNAS Scale

Free, open-source, ZFS-based. Rock-solid storage. Also supports containers and VMs. More opinionated than Unraid. Great choice if you prefer open-source and don't mind a steeper setup.

🟡

Synology DSM

The easiest. Polished interface, everything just works. Limited by the Synology hardware and proprietary ecosystem. Fine for simple setups; frustrating if you outgrow it.

Why I'd still pick Unraid

The mixed-drive flexibility, the Community Applications store, and the UI are genuinely hard to beat for a home setup. The licensing cost is the only real downside — and at $249 lifetime, it's a one-time expense that pays for itself fast.
